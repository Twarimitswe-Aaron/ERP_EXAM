# Rwanda Government ERP - Payroll Management System

An Enterprise-grade Spring Boot backend built to the exact specifications of the Rwanda ERP Payroll Exam. It handles multi-tenancy (Institutions), automated tax deductions, and database-level messaging triggers.

## Environment & Setup

1. **Prerequisites:**
   - Java 17+
   - Maven
   - Docker & Docker Compose (or local PostgreSQL)

2. **Starting the Application:**
   Run the following command to start PostgreSQL and the Application:
   ```bash
   docker-compose up -d
   ./mvnw spring-boot:run
   ```
   > The application runs on `http://localhost:8080`.

3. **Database Seeder (Super Admin):**
   When the application starts, it will automatically inject a `SUPER_ADMIN` into the database for immediate testing.
   - **Email:** `admin@gov.rw`
   - **Password:** `Admin@123` *(Validates: Min 5, Upper, Lower, Number, Special)*

## Testing via Swagger UI
This project comes with full OpenAPI Documentation.
1. Navigate to: [http://localhost:8080/swagger-ui/index.html](http://localhost:8080/swagger-ui/index.html)
2. Open the **Authentication** section and use `POST /api/auth/login` with `admin@gov.rw` / `Admin@123`.
3. Copy the returned `token` string.
4. Click the green **Authorize** button at the top of Swagger and paste the token as `Bearer <your_token>`.
5. You can now test any protected endpoint!

## API Endpoints List

### Authentication
- `POST /api/auth/register`: Register a new standard EMPLOYEE profile. Requires strict password validation.
- `POST /api/auth/login`: Authenticate and receive a JWT Bearer token.

### Employee & HR Management
- `POST /api/employees`: (Requires ADMIN) Create an Employee profile.
- `GET /api/employees`: View the company directory.
- `PUT /api/employees/{id}/employment`: (Requires ADMIN) Update an employee's department, position, and base salary.

### Deductions Configuration
- `POST /api/deductions`: (Requires ADMIN) Set global deduction percentages (Tax, Pension, Medical, etc.).
- `GET /api/deductions`: View current active deduction rules.

### Payroll Engine
- `POST /api/payroll/generate`: (Requires ADMIN) Core Engine Endpoint. Trigger this at the end of the month `{"month": 6, "year": 2026}`. The system automatically calculates gross salary, exact deductions, checks the `45%` limit rule, saves the payslips, and fires the PostgreSQL `BEFORE INSERT` trigger to generate the notification message.
- `GET /api/payroll/my-payslips`: Employees can view their historical generated payslips.

### Messaging & Notifications
- `GET /api/messages/my-messages`: Employees can fetch all 'Paid' notifications generated by the database trigger for their specific account.

## Security Features
- **Strict Password Enforcement:** Passwords must contain an Uppercase, Lowercase, Number, Special Character, and be at least 5 characters long.
- **Actuator Endpoints:** Publicly exposes `/actuator/health` and `/actuator/info`. All other metrics (e.g. `/actuator/env`) strictly require the `ADMIN` role.
- **Environment Variables:** Critical secrets like `JWT_SECRET` are pulled securely from the `.env` file instead of being hardcoded.
